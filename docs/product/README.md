### 2. game-mechanics.md（游戏机制详细设计）
**文档大小**：15KB
**文档概述**：详细设计游戏的各个系统机制，为开发 Agent 提供技术需求。

**主要内容**：
- **属性系统**：6 大基础属性 + 特殊属性（血继限界、仙人模式、封印术）
- **战斗系统**：战斗流程、行动类型、战斗状态、战斗 AI
- **忍术系统**：忍术等级、忍术分类、忍术学习
- **伙伴系统**：伙伴类型、伙伴属性、伙伴互动、伙伴羁绊
- **任务系统**：任务分类、任务难度、任务奖励
- **重生系统**：重生触发条件、重生继承、重生加成、重生奖励
- **经济系统**：货币类型、道具类型、市场系统
- **社交系统**：好友系统、公会/家族系统、师徒系统、婚姻系统
- **成就系统**：成就分类、成就奖励
- **排行榜系统**：排行榜类型、排行榜奖励
- **技能树系统**：技能树分类、技能点获取、技能前置
- **PVP 系统**：竞技场、野外 PVP、家族战
- **剧情系统**：剧情分支、影响因素、多结局系统、记忆系统
- **装备系统**：装备品质、装备强化、装备镶嵌、装备套装
- **存档系统**：存档类型、存档时机、存档内容

### 3. version-plan.md（版本规划和里程碑）
**文档大小**：12KB
**文档概述**：规划游戏从原型到正式上线的各个版本和开发里程碑。

**主要内容**：
- **开发路线图**：V0.1 → V0.2 → V0.5 → V0.9 → V1.0（共 17 周）
- **V0.1 - 核心原型**（2 周）：基础游戏框架、重生系统、剧情系统、数据存储
- **V0.2 - 功能扩展**（2 周）：战斗系统、忍术系统、伙伴系统、任务系统、经济系统
- **V0.5 - 内测版本**（4 周）：完整核心系统、剧情系统、伙伴系统、装备系统、血继限界系统、成就系统、排行榜系统
- **V0.9 - 公测准备**（3 周）：用户体验优化、社交系统、商业化系统、内容扩展、技术优化
- **V1.0 - 正式上线**（6 周）：完整游戏体验、高级功能、活动系统、运营功能、客户端
- **后续版本规划**：V1.1、V1.2、V1.5、V2.0
- **开发里程碑**：5 个关键里程碑及验收标准
- **资源分配**：人力资源、时间分配
- **风险管理**：技术风险、进度风险、运营风险
- **成功指标**：上线初期、稳定期、长期目标

### 4. retention-logic.md（留存和游戏节奏设计）
**文档大小**：17KB
**文档概述**：设计游戏的留存策略和节奏，提升用户长期活跃度。

**主要内容**：
- **留存策略概述**：留存目标、留存驱动力
- **用户生命周期管理**：
  - 新用户引导期（0-24 小时）
  - 沉浸期（24 小时 - 7 天）
  - 成长期（7-30 天）
  - 成熟期（30-90 天）
  - 留存维护期（90 天+）
- **游戏节奏设计**：
  - 节奏曲线（快速成长 → 稳定成长 → 慢速成长 → 收集完善）
  - 成长速度设计
  - 日常节奏、周常节奏、月常节奏、季度节奏
- **留存工具设计**：推送通知、邮件系统、礼包系统、补偿机制
- **流失预防**：流失预警指标、流失干预措施、回归机制
- **社交留存**：社交绑定设计、社交活动设计
- **数据分析与优化**：核心监控指标、用户分群分析、A/B 测试计划
- **长期运营策略**：内容更新节奏、活动运营策略、社区运营

## 使用指南

### 对于开发 Agent
1. **首先阅读** `game-concept.md`，了解游戏整体概念
2. **详细研读** `game-mechanics.md`，理解各个系统的设计细节
3. **参考** `version-plan.md`，了解开发阶段和里程碑
4. **关注** `retention-logic.md`，了解留存需求和节奏要求

### 对于 UI 设计师
1. **首先阅读** `game-concept.md`，了解游戏风格和目标用户
2. **参考** `game-mechanics.md`，了解各个界面的功能需求
3. **关注** `retention-logic.md`，了解用户引导和交互流程

3. **参考** `retention-logic.md`，了解用户引导节奏
